# 功能规格说明：在线实时 Markdown 协同编辑系统

**功能分支**：`001-md-realtime-collab`
**创建日期**：2026-04-18
**状态**：草稿
**输入**："构建一个 markdown 文件的在线实时协同编辑系统"

## 用户场景与测试

### 用户故事 1 — 注册与登录（优先级：P1）

新用户可以使用用户名、电子邮箱、电话号码和密码创建账户，然后登录系统访问各项功能。用户遗忘密码时也可以恢复访问权限。

**优先级原因**：身份认证是所有其他功能的入口——没有它什么都无法访问。

**独立测试**：用户使用有效凭证注册、收到确认、成功使用正确凭证登录，并通过邮箱/电话恢复遗忘的密码。

**验收场景**：

1. **假设** 我是一名新用户，**当** 我提交唯一的用户名、有效的电子邮箱、电话号码和匹配的密码，**那么** 我的账户被创建并自动登录。
2. **假设** 我拥有一个账户，**当** 我使用正确的用户名/邮箱/电话和密码登录，**那么** 我进入系统并看到我的工作空间。
3. **假设** 我已在一个设备上登录，**当** 我尝试从另一个设备登录，**那么** 第一个会话被注销，我必须在第一个设备上重新认证。
4. **假设** 我正用一个设备在活跃编辑文档，**当** 我的账户从另一个设备被登录，**那么** 系统在第一个设备上提示我保存未提交的变更，然后执行会话切换。
5. **假设** 我忘记了密码，**当** 我请求密码重置，**那么** 我收到重置链接/验证码并可以设置新密码。

---

### 用户故事 2 — 实时协同 Markdown 编辑（优先级：P1）

多个用户可以同时编辑同一个 Markdown 文件，实时看到彼此的变更。系统自动保存变更，无需手动操作。

**优先级原因**：这是系统的核心价值主张——实时协作。

**独立测试**：两个用户在不同浏览器中打开同一文件；其中一人的变更在 3 秒内出现在另一人的屏幕上。

**验收场景**：

1. **假设** 我对某文件拥有读写权限，**当** 我以编辑模式打开它，**那么** 其他同样有权限的用户能实时看到我的光标位置和变更。
2. **假设** 我对某文件只有只读权限，**当** 我打开它，**那么** 我能看到内容和预览，但无法编辑。
3. **假设** 我正在编辑一个文件，**当** 我的变更发生时，**那么** 系统自动保存，无需我点击保存按钮。
4. **假设** 我只想专注写作，**当** 我切换到纯编辑模式，**那么** 不显示预览面板，我拥有最大的编辑空间。
5. **假设** 我只想查看渲染后的输出，**当** 我切换到纯预览模式，**那么** 我看到渲染后的 Markdown，但看不到编辑器。
6. **假设** 我想同时看到两者，**当** 我切换到双栏模式，**那么** 编辑器和预览并排显示。
7. **假设** 我正在编辑器中输入，**当** 我在长文档中滚动，**那么** 行号可见，Markdown 语法有高亮显示。

---

### 用户故事 3 — 文件与目录管理（优先级：P1）

用户可以通过可折叠的树形侧边栏创建、组织、上传、下载、移动、复制和删除 Markdown 文件及目录。

**优先级原因**：没有文件管理，用户无法组织或持久化自己的工作。

**独立测试**：用户创建一个新文件、上传一个 Markdown 文件、下载它、将移动到新文件夹、然后删除它——每一步操作都立即反映在文件树中。

**验收场景**：

1. **假设** 我在我的工作空间中，**当** 我点击"新建文件"或"新建文件夹"，**那么** 文件树中出现一个新条目。
2. **假设** 文件夹中有文件，**当** 我点击侧边栏的折叠按钮，**那么** 侧边栏隐藏，我有更多屏幕空间用于编辑器。
3. **假设** 我有一个本地的 Markdown 文件，**当** 我上传它，**那么** 它出现在文件树中当前目录下。
4. **假设** 我选中了一个文件，**当** 我点击下载，**那么** 浏览器下载该 Markdown 文件及其相关资源。
5. **假设** 我是文件或文件夹的所有者，**当** 我选择移动/复制/删除，**那么** 操作完成，文件树更新。

---

### 用户故事 4 — 联系人管理（优先级：P2）

用户可以搜索其他用户、发送联系人邀请，并管理自己的联系人列表。邀请在 24 小时后过期。

**优先级原因**：协作需要联系人列表才能分享文件。

**独立测试**：用户 A 通过用户名搜索到用户 B，发送邀请，用户 B 收到并接受邀请，双方都在各自的联系人列表中看到对方。

**验收场景**：

1. **假设** 我已登录，**当** 我在搜索框输入用户名、邮箱或电话，**那么** 匹配的用户出现在结果中。
2. **假设** 我找到了想要协作的用户，**当** 我点击"添加联系人"，**那么** 邀请被发送给对方，我收到待处理状态通知。
3. **假设** 我收到一个联系人邀请，**当** 我在 24 小时内打开它，**那么** 我可以接受或拒绝；接受后双方添加到彼此的联系人列表，拒绝后邀请被移除。
4. **假设** 我收到一个联系人邀请，**当** 24 小时过去而我没有响应，**那么** 邀请过期，无法被接受。
5. **假设** 我有一个联系人，**当** 我选择移除他们，**那么** 他们从我的列表中被移除，我也从他们的列表中被移除。

---

### 用户故事 5 — 文件访问权限管理（优先级：P1）

文件所有者可以向选定的联系人授予、调整和撤销文件的只读或读写权限。权限变更立即通知被影响的用户。

**优先级原因**：权限管理是受控协作的核心——也就是核心分享机制。

**独立测试**：所有者向联系人 A 授予读写权限，向联系人 B 授予只读权限；A 可以编辑而 B 不能；所有者后来将 B 改为读写权限并撤销了 A——每种状态变更都正确执行。

**验收场景**：

1. **假设** 我是文件所有者，**当** 我选择一个或多个联系人并选择"授予读写权限"，**那么** 这些联系人可以编辑文件并看到实时更新。
2. **假设** 我是文件所有者，**当** 我选择一个或多个联系人并选择"授予只读权限"，**那么** 这些联系人可以查看和预览，但不能编辑。
3. **假设** 我拥有某文件的访问权限，**当** 所有者变更了我的权限级别，**那么** 我收到变更通知。
4. **假设** 我是文件所有者，**当** 我选择联系人并点击"撤销访问权限"，**那么** 这些联系人失去所有访问权限并收到通知。
5. **假设** 我正用读写权限编辑文件，**当** 所有者撤销了我的访问权限，**那么** 弹出一个对话框询问我是否保存未提交的变更；确认后我被重定向到主页。
6. **假设** 我正用读写权限编辑文件，**当** 所有者把我的权限改为只读，**那么** 弹出一个对话框询问我是否保存未提交的变更；确认后我留在页面中处于只读模式。

### 用户故事 6 — 用户资料与设置（优先级：P2）

用户可以更新个人资料信息和偏好设置，包括用户名、电子邮箱、电话号码和密码，以及主题和语言设置。

**优先级原因**：用户需要维护准确的联系方式和舒适的工作环境。

**独立测试**：用户成功修改用户名、邮箱和密码；在浅色和深色主题之间切换，在中文和英文之间切换——所有变更在会话间持久化。

**验收场景**：

1. **假设** 我已登录，**当** 我打开个人资料设置，**那么** 我可以修改我的用户名、邮箱、电话和密码。
2. **假设** 我修改用户名，**当** 新用户名未被占用，**那么** 变更被保存并处处反映。
3. **假设** 我修改密码，**当** 我提供正确的当前密码和匹配的新密码，**那么** 密码被更新。
4. **假设** 我在设置页面，**当** 我切换主题，**那么** 界面立即在浅色和深色模式之间切换。
5. **假设** 我在设置页面，**当** 我切换语言，**那么** 所有界面文本在中文和英文之间切换。

## 需求

### 功能需求

#### 认证与会话

- **FR-001**：系统必须允许用户使用用户名（唯一）、电子邮箱（唯一）、电话号码（唯一）、密码和确认密码进行注册。
- **FR-002**：系统必须强制密码复杂度：最少 12 个字符，至少一个特殊字符、一个大写字母、一个小写字母和一个数字。
- **FR-003**：系统必须允许用户使用用户名、电子邮箱或电话号码加密码登录。
- **FR-004**：系统必须支持双令牌认证（访问令牌 + 刷新令牌）；访问令牌 15 分钟后过期；刷新令牌 7 天后过期。

  **实现说明**：WebSocket 协作连接使用 access token，客户端每 30 秒检测一次 token 剩余有效期，当剩余时间少于 1 分钟时自动调用刷新接口获取新 token，通过 Yjs awareness 心跳机制通知协作服务器更新认证状态，确保长连接不因 token 过期而中断。
- **FR-005**：系统必须强制单会话模式：同一时间每个用户只能有一个活跃会话。
- **FR-006**：系统必须通过发送到用户邮箱或电话的限时验证码/链接提供密码重置功能。
- **FR-007**：系统必须允许用户在登录后修改用户名、电子邮箱、电话号码和密码。
- **FR-007b**：所有 API 错误响应必须使用统一格式 {code, message, details}；错误消息必须支持中英文本地化；速率限制必须按用户级别执行，超限返回 HTTP 429 状态码。

#### Markdown 编辑器

- **FR-008**：编辑器必须支持三种视图模式：纯编辑、纯预览和双栏（编辑 + 预览）。
- **FR-009**：编辑器必须支持完整的标准 Markdown 语法和常用扩展语法（表格、任务列表、脚注、删除线、表情符号）。
- **FR-010**：编辑器必须显示行号并应用语法高亮。
- **FR-011**：预览必须以等效的视觉保真度渲染编辑器支持的所有 Markdown 语法。
- **FR-012**：编辑器必须支持实时多用户协作：并发编辑在 3 秒内同步。
- **FR-013**：系统必须自动保存文件变更，无需用户干预；必须支持离线编辑，断网期间的变更在本地排队，恢复连接后与服务器状态合并，不丢失数据。

#### 文件管理

- **FR-014**：~~系统必须提供可折叠的侧边栏，显示文件和目录的树形结构。~~ **已移除**（2026-06）：folders 功能不再实现，文档列表使用扁平列表。
- **FR-015**：~~用户必须能够创建、重命名、移动、复制和删除文件及目录。~~ **已移除**（2026-06）：目录管理功能移除，保留文档创建/复制/删除。
- **FR-016**：用户必须能够上传（最大 50MB）和下载 Markdown 文件及相关资源文件。
- **FR-017**：~~文件操作必须立即反映在文件树中。~~ **已移除**（2026-06）：文件树功能移除。

#### 联系人管理

- **FR-018**：用户必须能够通过用户名、电子邮箱或电话号码搜索其他用户。
- **FR-019**：用户必须能够向另一个用户发送联系人邀请。
- **FR-020**：被邀请的用户必须能够接受或拒绝联系人邀请。
- **FR-021**：如果 24 小时内未响应，联系人邀请必须过期。
- **FR-022**：用户必须能够从联系人列表中移除联系人。

#### 权限管理

- **FR-023**：文件所有者必须能够向联系人授予只读或读写权限。
- **FR-024**：文件所有者必须能够撤销联系人的访问权限。
- **FR-025**：文件所有者必须能够批量选择多个联系人，一次性授予、调整或撤销权限。
- **FR-026**：权限变更必须通知被影响的联系人；通知持续保留直至用户手动清除。
- **FR-027**：当用户在活跃编辑时权限被撤销或降级为只读，系统必须提示用户保存未提交的变更，然后再应用新的权限状态。

#### 用户界面/用户体验

- **FR-028**：系统必须支持浅色和深色主题模式，并允许用户切换。
- **FR-029**：系统必须支持中文和英文语言，并允许用户切换。
- **FR-030**：系统必须实现响应式设计，适配移动端和桌面端浏览器。
- **FR-030b**：系统必须为所有空状态显示描述性占位内容（例如"暂无文件——创建你的第一个文件"），并在数据加载过程中显示骨架屏。
- **FR-030c**：系统必须满足 WCAG AA 级别可访问性合规，包括色彩对比度至少 4.5:1、键盘导航支持、ARIA 标签和屏幕阅读器支持；部分关键组件（如编辑器交互区域）进一步追求 WCAG AAA 合规。

#### 非功能性需求

- **NFR-001**：系统必须为认证、协作、文件操作的每个关键步骤生成结构化日志。
- **NFR-002**：系统必须记录所有错误日志，包含错误类型、用户ID（若已认证）、时间戳和调用链路。
- **NFR-003**：系统必须收集并暴露性能指标，包括 API 响应时间和协作同步延迟。
- **NFR-004**：系统必须支持分布式追踪信号，覆盖从请求入口到各微服务的完整调用链路。
- **NFR-005**：系统必须支持至少 1000 并发用户、至少 100,000 个文档，并保证 API 至少支持 500 QPS。

### 关键实体

- **User（用户）**：系统的注册用户。属性：id、username（用户名）、email（邮箱）、phone（电话）、passwordHash（argon2id 密码哈希，兼容 bcrypt 遗留哈希验证）、createdAt（创建时间）、updatedAt（更新时间）。
- **Document（文档）**：一个 Markdown 文件。属性：id、title（标题）、content（内容）、ownerId（所有者ID）、createdAt、updatedAt。
- ~~**Folder（文件夹）**~~：**已移除**（2026-06）。
- **Contact（联系人）**：两个用户之间的双向连接。属性：id、userId、contactUserId、createdAt。
- **ContactInvitation（联系人邀请）**：一个待处理的联系人请求。属性：id、inviterId（邀请人ID）、inviteeId（被邀请人ID）、status（状态）、expiresAt（过期时间）、createdAt。
- **Permission（权限）**：用户对文档的访问级别。属性：id、documentId、userId、level（read-only | read-write）、grantedBy（授权人）、createdAt、updatedAt。
- **Notification（通知）**：关于权限变更或其他事件的用户消息。属性：id、userId、type（类型）、content（内容）、read（已读）、createdAt。
- **Session（会话）**：一个活跃的用户会话。属性：id、userId、accessToken（访问令牌）、refreshToken（刷新令牌）、deviceInfo（设备信息）、createdAt、expiresAt。

## 成功标准

### 认证与协作

- **SC-001**：100% 的新用户能在 2 分钟内完成注册。
- **SC-002**：用户提交凭证后 5 秒内可以登录并获得有效会话。
- **SC-003**：5 名以上用户同时编辑同一文档，变更在 3 秒内同步。
- **SC-004**：任何 30 秒窗口内自动保存不丢失数据，且离线变更在恢复连接后正确合并。

### 文件管理

- **SC-005**：文件树操作（创建、重命名、移动、删除）在 1 秒内反映到界面。
- **SC-006**：用户可以成功上传最大 50MB 的文件。
- **SC-007**：用户可以下载其拥有至少只读权限的任何文件。

### 联系人与权限

- **SC-008**：联系人搜索在 2 秒内返回结果。
- **SC-009**：邀请在创建后恰好 24 小时过期并不可用。
- **SC-010**：权限变更通知在 5 秒内触达被影响的用户。

### 可用性

- **SC-011**：主题和语言偏好跨会话和设备持久化。
- **SC-012**：所有核心工作流（编辑、预览、文件管理、分享）在 375px 宽的移动屏幕上完全可用。
- **SC-013**：95% 的用户能在首次登录后 5 分钟内完成第一次文件编辑和分享。

## 假设条件

- 实时协作使用 WebSocket 结合 OT（操作转换）或 CRDT 实现无冲突并发编辑，并支持通过本地变更排队和恢复连接后同步的离线编辑。
- 文件内容和元数据存储在 PostgreSQL 中；实际文件资产可能引用外部对象存储。
- 通知通过应用内消息传递，无需外部推送通知基础设施。
- 搜索端点应用速率限制以防止枚举攻击。
- 单会话策略在服务器端强制执行：创建新会话时，该用户的所有先前会话均失效。
- 单个用户存储无上限；单个文件最大 50MB。
- 密码重置通过自建 SMTP 邮件服务器发送验证码，仅支持邮箱验证，暂不支持短信验证。
- 不允许访客或匿名访问；所有用户必须注册和认证；只有已认证用户可以访问任何功能。
